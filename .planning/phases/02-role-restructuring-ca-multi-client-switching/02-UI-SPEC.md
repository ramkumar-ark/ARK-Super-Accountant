## Spacing Scale

All values are multiples of 4. Tokens map to Tailwind CSS 4 spacing utilities.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon-to-label gaps, badge internal padding |
| sm | 8px | Button vertical padding, inline element gaps |
| md | 16px | Default element spacing, card inner padding |
| lg | 24px | Section padding, page content padding |
| xl | 32px | Between major layout sections |
| 2xl | 48px | Page top padding, major section breaks |
| 3xl | 64px | Topbar height (fixed), full-page vertical rhythm |

Exceptions:
- Touch targets: minimum 44px height on all interactive elements (buttons, role badge chips, org selector trigger, nav items). Source: design-system/MASTER.md.
- Topbar: exactly 64px height (CSS variable `--topbar-height`).
- Sidebar: 240px wide expanded, 64px collapsed. Source: index.css `--sidebar-width`, `--sidebar-collapsed-width`.
- Org selector dropdown: `max-h-80` (320px) with `overflow-y-auto` — already implemented in OrganizationSelector.tsx.

**authStore update required (not UI but affects UI behavior):** `switchOrganization` action must update `user.role` with the role returned from the org-switch endpoint, so the visibility condition above reflects the correct role for the newly active org. Source: RESEARCH.md Pattern 3.

## Interaction Contracts

### Org Switch Flow (AUDITOR_CA only)

1. User clicks org selector trigger → dropdown opens immediately, no loading state.
2. User clicks "Switch to [org name]" → row shows "Switching…" text, all other rows remain interactive.
3. API POST `/api/organizations/{id}/select` resolves → dropdown closes, trigger button updates to new org name, topbar role badge updates to new org's role.
4. API fails → "Switching…" replaced with X icon + error message inline in the failed row. Dropdown stays open. Other org rows remain interactive.
5. Clicking outside the open dropdown closes it (via `onClick` outside or `blur` event — use existing `open` state pattern from OrganizationSelector.tsx).

### Signup Role Change

1. User opens role select.
2. If arriving via `?invite=` token: select is disabled (`disabled` attribute), value is pre-filled from invite context. Visual: `opacity-50` (existing pattern).
3. If arriving without invite token: select is fully interactive, default value is `operator`.
4. No confirmation step on role change in signup — it is a pre-submission field.

### Role Badge — No Interaction

The role badge in the topbar is display-only. No click, no tooltip, no hover state beyond the default cursor. It is not a button. `role="status"` aria attribute to announce role to screen readers.

## Accessibility

- All interactive elements: minimum 44px touch target height.
- Org selector dropdown: `role="listbox"`, each row `role="option"` with `aria-selected`. Already implemented — preserve.
- Role badge: `role="status"` aria attribute, `aria-label="Your role: [role label]"`.
- Sidebar nav items: `aria-current="page"` on active item.
- Focus management on org switch: after dropdown closes on successful switch, return focus to the trigger button.
- Color contrast: all text/background combinations use existing tokens which meet WCAG AA (verified: `#0F172A` on `#FFFFFF` = 19:1, `#2563EB` on `#FFFFFF` = 5.9:1).
